# デフォルトシェーダーの後に実行するカスタムシェーダー。これはすべてのプラットフォーム向けの
# GLSL構文のシェーダーへのファイルパスです。
#
# 警告: 無効なシェーダーはGhosttyを使用不可にする可能性があります。たとえば、
# ウィンドウが完全に黒くなることがあります。このような場合は、このシェーダーを
# 無効にするためにこの設定を解除できます。
#
# カスタムシェーダーサポートはShadertoyシェーダーに基づいており、互換性があります。
# シェーダーは`mainImage`関数を指定する必要があり、利用可能なuniformは
# 大部分がShadertoyと一致していますが、いくつかの注意点とGhostty固有の拡張があります。
#
# シェーダーで利用可能なuniform値は以下の通りです：
#
#  * `sampler2D iChannel0` - 入力テクスチャ。
#
#    現在のターミナルスクリーンを含むテクスチャ。複数のカスタムシェーダーが
#    指定されている場合、前のシェーダーの出力がこのテクスチャに書き込まれます。
#    これにより複数の効果を組み合わせることができます。
#
#  * `vec3 iResolution` - 出力テクスチャサイズ、`[width, height, 1]`（ピクセル単位）。
#
#  * `float iTime` - 最初のフレームがレンダリングされてからの経過時間（秒）。
#
#  * `float iTimeDelta` - 前のフレームがレンダリングされてからの経過時間（秒）。
#
#  * `float iFrameRate` - 平均フレームレート。(現在サポートされていません)
#
#  * `int iFrame` - これまでにレンダリングされたフレーム数。
#
#  * `float iChannelTime[4]` - ビデオまたはサウンド入力の現在時刻。(該当なし)
#
#  * `vec3 iChannelResolution[4]` - 4つの入力サンプラーの解像度。
#
#    現在`iChannel0`のみが存在し、`iChannelResolution[0]`は
#    `iResolution`と同じです。
#
#  * `vec4 iMouse` - マウス入力情報。(現在サポートされていません)
#
#  * `vec4 iDate` - 日付/時刻情報。(現在サポートされていません)
#
#  * `float iSampleRate` - オーディオのサンプリングレート。(該当なし)
#
# Ghostty固有の拡張：
#
#  * `vec4 iCurrentCursor` - ターミナルカーソルに関する情報。
#
#    - `iCurrentCursor.xy`は現在のカーソルの-X, +Yコーナーです。
#    - `iCurrentCursor.zw`は現在のカーソルの幅と高さです。
#
#  * `vec4 iPreviousCursor` - 前のターミナルカーソルに関する情報。
#
#  * `vec4 iCurrentCursorColor` - ターミナルカーソルの色。
#
#  * `vec4 iPreviousCursorColor` - 前のターミナルカーソルの色。
#
#  * `float iTimeCursorChange` - ターミナルカーソル変更のタイムスタンプ。
#
#    ターミナルカーソルの位置または色が変更されると、これは
#    `iTime` uniformと同じ時刻に設定され、`iTime`からこれを引くことで
#    変更からの経過時間を計算できます。
#
# シェーダーのコンパイルに失敗した場合、そのシェーダーは無視されます。
# シェーダーコンパイル関連のエラーは設定エラーとして表示されず、
# ログにのみ表示されます。シェーダーコンパイルは設定読み込み後の
# 専用レンダリングスレッドで行われるためです。対話的な開発には、
# [shadertoy.com](https://shadertoy.com)を使用してください。
#
# これは複数回指定して複数のシェーダーを読み込むことができます。シェーダーは
# 指定された順序で実行されます。
#
# これはランタイムで変更可能であり、すべてのオープンターミナルに影響します。
