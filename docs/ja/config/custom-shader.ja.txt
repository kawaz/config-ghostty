# デフォルト シェーダーの後に実行するカスタム シェーダー。
# これは、すべてのプラットフォーム用の GLSL 構文シェーダーへのファイル パスです。
#
# 警告: 無効なシェーダーは、ウィンドウが完全に黒くなるなどの場合により、
# Ghostty を使用できなくする可能性があります。これが発生した場合は、
# この設定の設定を解除してシェーダーを無効にできます。
#
# カスタム シェーダー サポートは Shadertoy シェーダーに基づいており、
# 互換性があります。シェーダーは `mainImage` 関数を指定する必要があり、
# 利用可能なユニフォームは大部分が Shadertoy と一致していますが、
# 注意点と Ghostty 固有の拡張があります。
#
# シェーダーで利用可能なユニフォーム値は次のとおりです:
#
#  * `sampler2D iChannel0` - 入力テクスチャ。
#    現在のターミナル スクリーンを含むテクスチャ。複数のカスタム
#    シェーダーが指定されている場合、前のシェーダーの出力がこのテクスチャに
#    書き込まれます。複数のエフェクトを組み合わせることができます。
#
#  * `vec3 iResolution` - 出力テクスチャ サイズ、`[width, height, 1]` (px 単位)。
#
#  * `float iTime` - 最初のフレームがレンダリングされてからの時間 (秒単位)。
#
#  * `float iTimeDelta` - 前のフレームがレンダリングされてからの時間 (秒単位)。
#
#  * `float iFrameRate` - 平均フレームレート。(現在サポートされていません)
#
#  * `int iFrame` - これまでレンダリングされたフレーム数。
#
#  * `float iChannelTime[4]` - ビデオまたはサウンド入力の現在の時間。(該当なし)
#
#  * `vec3 iChannelResolution[4]` - 4 つの入力サンプラーの解像度。
#    現在 `iChannel0` のみが存在し、`iChannelResolution[0]` は `iResolution` と同一です。
#
#  * `vec4 iMouse` - マウス入力情報。(現在サポートされていません)
#
#  * `vec4 iDate` - 日付/時刻情報。(現在サポートされていません)
#
#  * `float iSampleRate` - オーディオのサンプル レート。(該当なし)
#
# Ghostty 固有の拡張:
#
#  * `vec4 iCurrentCursor` - ターミナル カーソルに関する情報。
#    - `iCurrentCursor.xy` は現在のカーソルの -X、+Y コーナーです。
#    - `iCurrentCursor.zw` は現在のカーソルの幅と高さです。
#
#  * `vec4 iPreviousCursor` - 前のターミナル カーソルに関する情報。
#
#  * `vec4 iCurrentCursorColor` - ターミナル カーソルの色。
#
#  * `vec4 iPreviousCursorColor` - 前のターミナル カーソルの色。
#
#  * `float iTimeCursorChange` - ターミナル カーソル変更のタイムスタンプ。
#    ターミナル カーソルが位置または色を変更する場合、これは `iTime` ユニフォーム
#    と同じ時刻に設定され、`iTime` からこれを減算することで、変更以降の時間を計算できます。
#
# シェーダーのコンパイルに失敗した場合、シェーダーは無視されます。
# シェーダー コンパイルに関連するエラーは設定エラーとして表示されず、
# 専用レンダー スレッドで設定読み込み後にシェーダー コンパイルが発生するため、
# ログに表示されるのみです。対話的な開発には [shadertoy.com](https://shadertoy.com) を使用してください。
#
# これを複数回繰り返して、複数のシェーダーを読み込むことができます。
# シェーダーは指定された順序で実行されます。
#
