# config-file

**言語:** [English](../../en/config/config-file.en.md) | 日本語

## 説明

追加で読み込む設定ファイル。この設定は繰り返し使用でき、複数の設定ファイルを読み込める。設定ファイル自体が別の設定ファイルを読み込むことも可能。パスは `config-file` ディレクティブを含むファイルから相対的。コマンドライン引数の場合は現在の作業ディレクトリから相対的。

ファイルパスの前に `?` 文字を付けると、ファイルが存在しない場合のエラーを抑制できる。リテラル `?` で始まるファイルを含めたい場合は、ファイルパスをダブルクォーテーション (") で囲む。

サイクルは許可されない。サイクルが検出されると、エラーがログに記録され、設定ファイルは無視される。

設定ファイルは、その内容が定義されている設定の後に、定義されている順序で読み込まれる。**これは非常に微妙だが重要なポイント。** つまり、設定ファイルは全体の設定が読み込まれた後でのみ有効になる。例えば、以下の設定では：

```
config-file = "foo"
a = 1
```

"foo" に `a = 2` が含まれていた場合、最終的な `a` の値は 2 になる。なぜなら `foo` は `config-file` 値を設定する設定ファイルの後に読み込まれるから。

## デフォルト

```conf
# https://ghostty.org/docs/config/reference#config-file
# Ghostty 用の追加設定ファイルを読み込む (デフォルト: (なし))
config-file =
```
