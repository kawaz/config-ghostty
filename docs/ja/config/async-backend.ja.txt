# 非同期IO、イベント処理などに使用する低レベルAPIを設定します。
#
# ほとんどのユーザーは `auto` のままにしておくべきです。これにより自動的に
# APIが利用できない状況（例：特定のハードニングされたカーネルの `io_uring`）を
# 検出し、別のAPIにフォールバックします。ただし、何らかの理由で特定のバック
# エンドを強制したい場合は、ここで設定することができます。
#
# 様々なベンチマークに基づいて、バックエンド間でメモリ、CPU、または遅延に関して
# 統計的に有意な差を見つけていません。バックエンドの選択は、互換性と機能に関
# するものです。
#
# 利用可能なオプション：
#
#   * `auto` - プラットフォームに応じて利用可能なオプションに基づいて、
#     最適なバックエンドを自動的に選択します。
#   * `epoll` - `epoll` APIを使用します。
#   * `io_uring` - `io_uring` APIを使用します。
#
# 選択したバックエンドがプラットフォームで利用できない場合、Ghosttyは
# 利用可能な自動選択されたバックエンドにフォールバックします。
#
# この値を変更するには、アプリケーションの完全な再起動が必要です。
#
# これはLinuxでのみサポートされています。これは複数のオプションがある唯一の
# プラットフォームです。macOSでは、常に `kqueue` を使用します。
#
# サポート開始バージョン：1.2.0
async-backend = auto
