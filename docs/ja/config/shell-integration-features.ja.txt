# 有効にするシェル統合機能。これらはシェル統合を必要とします
# シェル統合は自動（shell-integration経由）またはマニュアルで読み込みます。
#
# このフォーマットは、カンマで区切られた有効にする機能のリストです。機能の前に
# `no-` を付けるとその機能は無効になります。機能を省略した場合、そのデフォルト値が
# 使用されるため、不要な機能は明示的に無効にする必要があります。また、`true` または
# `false` を使用してすべての機能のオン/オフを切り替えることもできます。
#
# 例: `cursor`, `no-cursor`, `sudo`, `no-sudo`, `title`, `no-title`
#
# 利用可能な機能:
#
#   * `cursor` - プロンプトでカーソルを点滅するバーに設定します。
#
#   * `sudo` - sudo ラッパーを設定して terminfo を保持します。
#
#   * `title` - シェル統合経由でウィンドウタイトルを設定します。
#
#   * `ssh-env` - SSH 環境変数の互換性を有効にします。リモートホストに接続する
#     際に TERM を `xterm-ghostty` から `xterm-256color` に自動変換し、
#     COLORTERM、TERM_PROGRAM、および TERM_PROGRAM_VERSION を伝播します。
#     これらの変数がリモートホストで受け入れられるかどうかは、
#     これらの変数が sshd_config で許可されているかどうかによって異なります。
#     (利用可能な時期: 1.2.0 以降)
#
#   * `ssh-terminfo` - リモートホストでの自動 terminfo インストールを有効にします。
#     terminfo がない ホストに接続する際、`infocmp` と `tic` を使用して
#     Ghostty の terminfo エントリをインストールしようとします。ローカルで
#     `infocmp` が利用可能で、リモートホストで `tic` が利用可能である必要があります。
#     terminfo がリモートホストにインストールされると、繰り返しインストールを
#     避けるために自動的に「キャッシュ」されます。必要に応じて、`+ssh-cache`
#     CLI アクションを使用してインストールキャッシュを様々な引数で手動管理できます。
#     (利用可能な時期: 1.2.0 以降)
#
#   * `path` - Ghostty のバイナリディレクトリを PATH に追加します。これにより
#     シェル初期化スクリプトが PATH をリセットしても、シェルで `ghostty` コマンドが
#     利用可能になります。これは PATH がシステムスクリプトによって上書きされることが
#     多い macOS で特に有用です。ディレクトリはまだ存在しない場合にのみ追加されます。
#
# SSH 機能は独立して動作でき、最適な体験のために組み合わせることができます:
# `ssh-env` と `ssh-terminfo` が両方有効になっている場合、Ghostty はリモートホストに
# terminfo をインストールし、TERM として `xterm-ghostty` を使用し、
# terminfo インストール失敗時には環境変数を使用して `xterm-256color` にフォールバックします。
shell-integration-features = cursor,no-sudo,title,no-ssh-env,no-ssh-terminfo,path
