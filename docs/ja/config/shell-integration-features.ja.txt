# 有効にするシェル統合機能。これらは、shell-integration 経由で
# 自動的に、または手動でロードされるシェル統合が必要である。
#
# これの形式はコンマで区切られた有効にする機能のリストである。
# 機能に `no-` プレフィックスを付けると、それは無効になる。
# 機能を省略すると、そのデフォルト値が使用される。そのため、
# 必要のない機能を明示的に無効にする必要がある。
# また、`true` または `false` を使用すると、すべての機能をオン/オフにできる。
#
# 例：`cursor`、`no-cursor`、`sudo`、`no-sudo`、`title`、`no-title`
#
# 利用可能な機能：
#
#   * `cursor` - プロンプトでカーソルを点滅するバーに設定する。
#
#   * `sudo` - sudo ラッパーを terminfo を保持するように設定する。
#
#   * `title` - シェル統合経由でウィンドウ タイトルを設定する。
#
#   * `ssh-env` - SSH 環境変数互換性を有効にする。リモート ホストに
#     接続するときに TERM を `xterm-ghostty` から `xterm-256color` に
#     自動的に変換し、COLORTERM、TERM_PROGRAM、および TERM_PROGRAM_VERSION
#     を伝播する。これらの変数がリモート ホストで受け入れられるかどうかは、
#     変数が sshd_config で許可されているかどうかによって異なる。
#     （1.2.0 以降で利用可能）
#
#   * `ssh-terminfo` - リモート ホストへの自動 terminfo インストールを有効にする。
#     それを欠くホストに接続するときに、`infocmp` と `tic` を使用して
#     Ghostty の terminfo エントリをインストールしようとする。
#     `infocmp` をローカルで利用でき、`tic` をリモート ホストで利用可能である
#     ことが必要である。terminfo がリモート ホストにインストールされると、
#     繰り返しのインストールを回避するために自動的に「キャッシュ」される。
#     必要に応じて、`+ssh-cache` CLI アクションを使用して、
#     さまざまな引数を使用してインストール キャッシュを手動で管理できる。
#     （1.2.0 以降で利用可能）
#
#   * `path` - Ghostty のバイナリ ディレクトリを PATH に追加する。
#     これにより、シェル init スクリプトが PATH をリセットしてもシェルで
#     `ghostty` コマンドが利用可能であることが保証される。
#     これは、PATH がシステム スクリプトによって上書きされることが多い
#     macOS で特に有用である。ディレクトリは、まだ存在していない場合にのみ追加される。
#
# SSH 機能は独立して機能し、最適な体験のために組み合わせることができる：
# `ssh-env` と `ssh-terminfo` の両方が有効になっている場合、
# Ghostty はリモート ホストに terminfo をインストールし、TERM として
# `xterm-ghostty` を使用し、terminfo のインストールが失敗した場合は
# 環境変数で `xterm-256color` にフォールバックする。
shell-integration-features = cursor,no-sudo,title,no-ssh-env,no-ssh-terminfo,path
