# スクロールバックバッファのサイズ（バイト単位）。これはアクティブな
# スクリーンも含みます。この設定値がどうあれ、表示可能なスクリーン用の
# メモリは常に確保され、残りがスクロールバックの制限となります。
#
# この制限に達すると、最も古い行がスクロールバックから削除されます。
#
# スクロールバックは現在、完全にメモリ上に存在します。つまり、
# この値が大きいほど、メモリ使用量の可能性が高くなります。
# スクロールバックはこの制限までの間、遅延アロケーション（lazy allocation）
# されるため、非常に大きな値に設定しても、すぐには大量のメモリを
# 消費しません。
#
# このサイズはターミナルサーフェス単位であり、アプリケーション全体ではありません。
#
# 現在のところ、無制限のスクロールバックバッファを設定することは
# できません。これは今後予定されている機能です。
#
# これは実行時に変更できますが、新しいターミナルサーフェスにのみ
# 影響します。
scrollback-limit = 10000000
