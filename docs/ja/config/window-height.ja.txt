# 初期ウィンドウサイズ。このサイズはデフォルトではターミナルグリッドセルで指定されます。
# 両方の値を設定して初めて有効になります。値が1つだけ設定されている場合は、
# それは無視されます。
#
# 現在ピクセル単位でのサイズ指定はサポートされていませんが、
# 将来の変更で対応予定です。指定されない場合は、アプリケーション
# ランタイムがデフォルトサイズを決定します。
#
# ウィンドウマネージャーがサイズに制限を設定したり、サイズを上書きする
# ことがあることに注意してください。例えば、タイリングウィンドウマネージャーは
# ウィンドウをグリッド内に収まるよう特定のサイズに強制することがあります。
# これについてGhosttyにできることはありませんが、最善の努力をします。
#
# スクリーンサイズより大きいサイズは、スクリーンサイズにクランプ
# されます。これを使用して、デフォルトで最大化されたウィンドウサイズを
# 作成できます。
#
# これは新しいタブ、分割、またはその他のネストされたターミナル要素には
# 影響を与えません。これは、任意の新しいウィンドウの初期ウィンドウサイズにのみ
# 影響を与えます。この値を変更しても、ウィンドウが作成された後のサイズは
# 変わりません。これは初期サイズにのみ使用されます。
#
# バグ: Linux（GTK）では、計算されたウィンドウサイズがウィンドウの装飾を
# 正しく考慮しません。その結果、グリッドの寸法がこの設定と完全には
# 一致しません。ウィンドウの装飾が無効化されている場合（「window-decoration」を参照）、
# これは期待通りに機能します。
#
# 幅10未満、高さ4未満のウィンドウは許可されていません。
window-height = 0
