# 起動時にコマンドに送信する入力データ。
#
# 設定された `command` は通常のルールを使用して起動され、
# その後、この入力として指定されたデータが pty に書き込まれます。
# その他の入力が提供される前に。
#
# バイトはそのまま送信され、追加のエンコーディングは行われません。したがって、
# 制御文字を含む可能性のある入力には注意してください。これは
# シェル内でプログラムを実行するために使用される可能性があります。
#
# このデータの形式は以下の通りです：
#
#   * `raw:<string>` - 生のテキストをそのまま送信します。これは Zig 文字列リテラル
#     構文を使用するため、制御文字やその他の標準的な
#     エスケープを指定できます。
#
#   * `path:<path>` - ファイルパスを読み込み、その内容を送信します。パスは
#     長さが有限なファイルを指す必要があります。例えば、
#     `/dev/stdin` や `/dev/urandom` などのデバイスを使用しないでください。
#     これらはターミナルの起動を無期限にブロックします。
#     ファイルのサイズは最大 10MB に制限されており、
#     過度なメモリ使用を防止しています。これより大きいファイルがある場合は、
#     ファイルを読み込みターミナルに送信するスクリプトを書く必要があります。
#
# 有効なプレフィックスが見つからない場合、`raw:` 入力と見なされます。
# これはエルゴノミック上の選択であり、単に
# `input = "Hello, world!"` （一般的なケース）と記述することができます。
# すべての値に `raw:` をプレフィックスとして付ける必要がありません。
#
# これを複数回繰り返して、より多くのデータを送信できます。
# データは直接連結され、区切り文字はありません。
# （例えば、改行はありません）。
#
# 入力ソースのいずれかが存在しない場合、入力はどれも
# 送信されません。入力ソースはターミナル起動時まで検証されないため、
# 不足したパスは設定検証に表示されません。
#
# 実行時にこの設定を変更すると、新しい
# ターミナルにのみ影響します。
#
