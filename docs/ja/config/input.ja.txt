# 起動時にコマンドへの入力として送信するデータ。
#
# 設定された `command` は典型的なルールを使用して起動され、
# この入力として指定されたデータは、他の入力が提供される前に
# pty に書き込まれる。
#
# バイトは追加エンコーディングなしでそのまま送信される。
# したがって、制御文字を含む可能性のある入力に注意してください。
# これはシェルでプログラムを実行するために使用できる。
#
# この値の形式は：
#
#   * `raw:<string>` - raw テキストをそのまま送信する。これは Zig 文字列
#     リテラル構文を使用しているため、制御文字と他の標準エスケープを
#     指定できる。
#
#   * `path:<path>` - ファイルパスを読み込み、内容を送信する。パスは
#     有限の長さのファイルへのパスである必要がある。例：`/dev/stdin`
#     または `/dev/urandom` などのデバイスを使用しないでください。
#     これらはターミナルの起動を無限に ブロックする。
#     ファイルは過度なメモリ使用を防ぐため、10MB のサイズに制限されている。
#     これより大きいファイルがある場合は、ファイルを読み込んで
#     ターミナルに送信するスクリプトを作成する必要がある。
#
# 有効なプレフィックスが見つからない場合、`raw:` 入力と想定される。
# これはエルゴノミックな選択であり、`raw:` プレフィックスで
# すべての値をプレフィックスする必要なく、
# `input = "Hello, world!"` （一般的な場合）を簡単に書くことができる。
#
# これを複数回繰り返して、より多くのデータを送信できる。
# データは、間に区切り文字なしで直接連結される（例：改行なし）。
#
# 入力ソースが存在しない場合、入力は送信されない。
# 入力ソースはターミナルの起動時まで検証されないため、
# 欠落しているパスは設定の検証に表示されない。
#
# 実行時にこの設定を変更すると、新しいターミナルのみに影響する。
#
