# `background-opacity` が 1 より小さい場合、背景をぼかすかどうか。
#
# 有効な値は：
#
#   * ぼかしの強度を指定する負でない整数
#   * `false`。ぼかし強度 0 に相当する
#   * `true`。デフォルトのぼかし強度 20 に相当し、見た目の良い
#     ぼかしに適している。より高いぼかし強度を使用するとレンダリング
#     の問題やパフォーマンスの問題が発生する可能性がある。
#
# macOS と一部の Linux デスクトップ環境でサポートされている：
#
#   * KDE Plasma（Wayland と X11）
#
# 警告：KDE Plasma では正確なぼかし強度は _無視_ される。この設定を
# `true` に設定するか、正の任意のぼかし強度値を使用すると、同じ効果が
# 得られる。理由は、Plasma を駆動するウィンドウコンポジター KWin には
# グローバルぼかし設定が1つしかなく、アプリケーションが個別のぼかし
# 設定を指定することを許可しないためである。
#
# KWin のグローバルぼかし設定を構成するには、システム設定を開き、
# 「アプリケーションとウィンドウ」>「ウィンドウ管理」>「デスクトップ効果」に移動して、
# 「ぼかし」プラグインを選択する。無効化されている場合は、左側のチェックボックスをチェックして有効化する。
# 次に「構成」ボタンをクリックすると、Ghostty を含むすべてのアプリケーションの
# 背景ぼかしとノイズ強度を設定することができる2つのスライダーが表示される。
#
# その他のすべての Linux デスクトップ環境は現在のところサポートされていない。
# 背景ぼかしをサポートするには、ユーザーが環境固有の設定を行うか、
# または統一されたインターフェースがないため、サードパーティプラグインをインストールする必要がある。
background-blur = false
