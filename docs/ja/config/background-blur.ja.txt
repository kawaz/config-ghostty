# `background-opacity` が 1 未満の場合、背景をぼかすかどうか。
#
# 有効な値：
#
#   * ぼかしの強度を指定する負でない整数
#   * `false`、ぼかし強度 0 に相当
#   * `true`、デフォルトのぼかし強度 20 に相当。これは良い
#     外観のぼかしに適した値です。より高いぼかし強度は
#     不可解なレンダリングとパフォーマンスの問題を
#     引き起こす可能性があります。
#
# macOS およびいくつかの Linux デスクトップ環境でサポートされています。
# サポートされている環境には以下が含まれます：
#
#   * KDE Plasma（Wayland および X11）
#
# 警告：KDE Plasma では正確なぼかし強度は _無視_ されます。
# この設定を `true` または任意の正のぼかし強度値に設定すると
# 同じ効果が得られます。理由は KWin（Plasma を駆動するウィンドウコンポジタ）は
# グローバルなぼかし設定のみを持ち、アプリケーションが
# 個別のぼかし設定を指定することを許可しないためです。
#
# KWin のグローバルぼかし設定を構成するには、システム設定を開いて
# 「アプリとウィンドウ」> 「ウィンドウ管理」> 「デスクトップエフェクト」に移動し、
# 「ぼかし」プラグインを選択します。無効な場合は、左側のチェックボックスを
# オンにして有効にします。「設定」ボタンをクリックすると、
# すべてのアプリ（Ghostty を含む）の背景ぼかしと
# ノイズ強度を設定できる 2 つのスライダーが表示されます。
#
# その他すべての Linux デスクトップ環境は現在サポートされていません。
# ユーザーは背景ぼかしをサポートするために、環境固有の設定を設定
# および/または サードパーティプラグインをインストールする必要があります。
# これを行うための統一されたインターフェースがないためです。
background-blur = false
