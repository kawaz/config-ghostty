# input

**言語:** [English](../../en/config/input.en.md) | 日本語

## 説明

起動時にコマンドへの入力として送信するデータを設定します。

設定された `command` は典型的なルールを使用して起動され、この入力として指定されたデータは、他の入力が提供される前に pty に書き込まれます。

バイトは追加エンコーディングなしでそのまま送信されます。したがって、制御文字を含む可能性のある入力に注意してください。これはシェルでプログラムを実行するために使用できます。

この値の形式は以下の通りです：

- `raw:<string>` - raw テキストをそのまま送信します。これは Zig 文字列リテラル構文を使用しているため、制御文字と他の標準エスケープを指定できます。

- `path:<path>` - ファイルパスを読み込み、内容を送信します。パスは有限の長さのファイルへのパスである必要があります。例：`/dev/stdin` または `/dev/urandom` などのデバイスを使用しないでください。これらはターミナルの起動を無限にブロックします。ファイルは過度なメモリ使用を防ぐため、10MB のサイズに制限されています。これより大きいファイルがある場合は、ファイルを読み込んでターミナルに送信するスクリプトを作成する必要があります。

有効なプレフィックスが見つからない場合、`raw:` 入力と想定されます。これはエルゴノミックな選択であり、`raw:` プレフィックスですべての値をプレフィックスする必要なく、`input = "Hello, world!"` （一般的な場合）を簡単に書くことができます。

これを複数回繰り返して、より多くのデータを送信できます。データは、間に区切り文字なしで直接連結されます（例：改行なし）。

入力ソースが存在しない場合、入力は送信されません。入力ソースはターミナルの起動時まで検証されないため、欠落しているパスは設定の検証に表示されません。

実行時にこの設定を変更すると、新しいターミナルのみに影響します。

利用可能：1.2.0 以降

## デフォルト値

```conf
# https://ghostty.org/docs/config/reference#input
# 起動時にコマンドに送信する入力データ (default: unset)
input =
```
