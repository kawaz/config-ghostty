# 使用するフォントファミリー。
#
# CLIを使用して有効な値のリストを生成できます。
#
#     ghostty +list-fonts
#
# この設定は複数回繰り返して指定でき、プライマリフォントで要求された
# コードポイントが利用できない場合の推奨フォールバックフォントを
# 指定するために使用されます。これは複数言語、記号フォント
# などの場合に特に便利です。
#
# 絵文字に関する注意：macOSではGhosttyはデフォルトで常にApple Color Emoji
# を使用し、Linuxでは常にNoto Emojiを使用します。font-familyここに
# 絵文字グリフを含むフォントファミリーを指定することで、この動作をオーバーライド
# できます。
#
# 特定のスタイル（ボールド、イタリック、ボールドイタリック）は明示的に
# 設定する必要はありません。スタイルが設定されていない場合、通常の
# スタイル（font-family）がスタイリストバリアントについて検索されます。
# スタイリストバリアントが見つからない場合、Ghosttyは通常のスタイルを使用
# します。これにより、ボールドなどのスタイルを取得するためだけに異なる
# フォントファミリーにフォールバックすることが防止されます。これは
# スタイル用に明示的にフォントファミリーを指定した場合にも適用されます。
# 例えば、`font-family-bold = FooBar`を設定し、"FooBar"が見つからない場合、
# Ghosttyはボールドスタイルに対して`font-family`に設定されている
# フォントを使用します。
#
# 最後に、サポートされていない場合、一部のスタイルは合成される可能性があります。
# 例えば、フォントがイタリックスタイルを持たず、代替イタリック
# フォントが指定されていない場合、Ghosttyは通常のスタイルにスラントを
# 適用することでイタリックスタイルを合成します。これらの合成された
# スタイルを無効にする場合は、以下で説明する`font-style`設定を使用
# できます。
#
# スタイルを完全に無効にするには、`font-style`設定セットを使用します。
# `font-style`のドキュメントを参照してください。
#
# 以前に設定された値を上書きしてフォールバックを追加する代わりに、
# 値を`""`（空文字列）として指定してリストをリセットしてから
# 新しい値を設定します。例えば：
#
#     font-family = ""
#     font-family = "My Favorite Font"
#
# これらのいずれかをCLI引数として設定すると、設定ファイルで設定された
# 値が自動的にクリアされるため、新しい値を設定する前に
# `--font-family=""`を指定する必要はありません。これは設定ファイル内
# で以前に設定された値をクリアしたい場合、またはCLI上で
# CLIで設定された値をクリアしたい場合に指定する必要があります。
