# フォント形成(シェーピング)を複数の実行に分割する場所。
#
# 「実行」(run)は、一緒に形成される連続したテキストセグメントです。「形成」
# (shaping)は、テキスト(コードポイント)をグリフ(レンダリング可能な文字)に
# 変換するプロセスです。これはリガチャ(合字)が形成される方法を含みます。
# たとえば、コーディングフォントが「!=」を単一のグリフに変換する場合、「!」
# と「=」が単一の実行内で隣同士に表示される必要があります。実行が分割されると、
# テキストは個別に形成されます。前述の例を続けると、「!」が1つの実行の終わりに
# あり、「=」が次の実行の開始にある場合、リガチャは形成されません。
#
# Ghosttyは、可読性またはユーザビリティを向上させるために、特定の点で実行を
# 分割します。たとえば、Ghosttyはデフォルトでカーソルの下の実行を分割するため、
# テキスト編集ではリガチャではなく個々の文字を認識できます。
# この設定では、この動作を構成できます。
#
# 複数のオプションを設定するには、値をコンマで結合します。オプションを無効にする
# には、「no-」を接頭辞として付けます。オプションの有効化と無効化は
# 同時に行うことができます。
#
# 利用可能なオプション:
#
#   * `cursor` - カーソルの下の実行を分割します。
#
# 対応バージョン: 1.2.0
font-shaping-break = cursor
