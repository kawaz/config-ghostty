# フォント シェーピングを複数の実行に分割する場所。
#
# 「実行」は、一緒に形成されるテキストの隣接するセグメントである。
# 「シェーピング」は、テキスト（コードポイント）をグリフ（レンダリング可能な文字）に
# 変換するプロセスである。これはリガチャがどのように形成されるかである。
# たとえば、コーディング フォントが「!=」を単一のグリフに変えた場合、
# 「!」と「=」が単一の実行で隣同士にいることを確認する必要がある。
# 実行が破損すると、テキストは個別に形成される。例を続けると、
# 「!」が 1 つの実行の最後にあり、「=」が次の実行の開始にある場合、
# リガチャは形成されない。
#
# Ghostty は可読性またはユーザビリティを改善するために、特定の点で実行を破損する。
# たとえば、Ghostty はデフォルトでカーソルの下で実行を破損して、
# テキスト編集がリガチャではなく個々の文字を確認できるようにする。
# この設定により、この動作を構成することができる。
#
# 複数のオプションを設定するには、コンマで値を結合する。
# オプションを無効にするには、「no-」をプレフィックスに付ける。
# オプションの有効化と無効化は同時に行うことができる。
#
# 利用可能なオプション：
#
#   * `cursor` - カーソルの下で実行を破損する。
#
# 利用可能：1.2.0 以降
font-shaping-break = cursor
