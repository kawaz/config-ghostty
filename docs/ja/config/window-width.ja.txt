# 初期ウィンドウサイズ。このサイズはデフォルトでターミナルグリッドセルで表されます。
# 効果を持たせるには両方の値を設定する必要があります。1つの値だけが設定されている場合、
# それは無視されます。
#
# 現在のところ、ピクセル単位でのサイズ指定はサポートされていませんが、
# 将来の変更でそれが可能になります。これが指定されていない場合、
# アプリランタイムがデフォルトサイズを決定します。
#
# ウィンドウマネージャーはサイズに制限を設けたり、サイズをオーバーライドしたり
# する可能性があることに注意してください。たとえば、タイリングウィンドウマネージャーは、
# グリッド内に適合するようにウィンドウを特定のサイズにするように強制する可能性があります。
# Ghostty がこれについて何かできることはありませんが、努力するでしょう。
#
# 画面サイズより大きいサイズはスクリーンサイズにクランプされます。
# これは、最大化されたデフォルトウィンドウサイズを作成するために使用できます。
#
# これは新しいタブ、スプリット、または他のネストされたターミナル要素には影響しません。
# これは、任意の新しいウィンドウの初期ウィンドウサイズにのみ影響します。
# このウィンドウが作成された後のウィンドウのサイズは変わりません。
# これは初期サイズにのみ使用されます。
#
# バグ：GTK を使用した Linux では、計算されたウィンドウサイズは
# ウィンドウの装飾を適切に考慮しません。その結果、グリッドの寸法が
# この設定と正確に一致しません。ウィンドウの装飾が無効になっている場合
# （`window-decoration` を参照）、これは期待どおりに機能します。
#
# 10 より小さい幅または 4 より小さい高さのウィンドウは許可されていません。
window-height = 0
window-width = 0
