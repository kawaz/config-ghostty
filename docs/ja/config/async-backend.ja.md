# async-backend

**言語:** [English](../../en/config/async-backend.en.md) | 日本語

## 説明

非同期 IO、イベント処理などに使用する低レベル API を設定します。

ほとんどのユーザーは、これを `auto` のままにしておく必要があります。これは、API が利用できない可能性があるシナリオを自動的に検出し（たとえば、特定のセキュリティ強化カーネル上の `io_uring`）、別の API にフォールバックします。ただし、何らかの理由で特定のバックエンドを強制したい場合は、ここで設定できます。

さまざまなベンチマークに基づいて、メモリ、CPU、またはレイテンシーに関して、バックエンド間の統計的に有意な違いは見つかりませんでした。バックエンドの選択は、互換性と機能に関するものがより多いです。

利用可能なオプション：

- `auto` - 利用可能なオプションに基づいて、プラットフォーム用の最良のバックエンドを自動的に選択します。
- `epoll` - `epoll` API を使用します
- `io_uring` - `io_uring` API を使用します

選択されたバックエンドがプラットフォームで利用できない場合、Ghostty は利用可能な自動的に選択されたバックエンドにフォールバックします。

この値を変更するには、フルアプリケーション再起動が必要です。

これは Linux でのみサポートされています。これは複数のオプションを持つ唯一のプラットフォームだからです。macOS では、常に `kqueue` を使用します。

利用可能な場合：1.2.0

## デフォルト

```conf
# https://ghostty.org/docs/config/reference#async-backend
# 非同期 IO とイベント処理に使用する低レベル API (default: auto)
async-backend = auto
```
