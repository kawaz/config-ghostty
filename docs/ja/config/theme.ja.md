# theme

**Language:** [English](../../en/config/theme.en.md) | 日本語

## Description

使用するテーマ。これは、組み込みテーマ名、カスタム テーマ名、またはカスタム テーマ ファイルへの絶対パスです。Ghostty は、ライトモードとダーク モード用に異なるテーマを指定することもサポートしています。各オプションについては以下に説明します。

テーマが絶対パス名の場合、Ghostty はそのファイルをテーマとして読み込もうとします。そのファイルが存在しないかアクセス不可の場合、エラーがログに記録され、他のディレクトリは検索されません。

テーマが絶対パス名でない場合、テーマと一致するファイル名について 2 つの異なるディレクトリが検索されます。これは、ファイルシステムが大文字と小文字を区別するシステムでは大文字と小文字を区別します。テーマ名に絶対パス名でない限り、パス セパレータを含めるのはエラーです。

最初のディレクトリは Ghostty 設定ディレクトリの `themes` サブディレクトリです。これは `$XDG_CONFIG_HOME/ghostty/themes` または `~/.config/ghostty/themes` です。

2 番目のディレクトリは Ghostty リソース ディレクトリの `themes` サブディレクトリです。Ghostty は多数のテーマを出荷しており、このディレクトリにインストールされます。macOS では、このリストは `Ghostty.app/Contents/Resources/ghostty/themes` ディレクトリにあります。Linux では、このリストは `share/ghostty/themes` ディレクトリにあります (Ghostty「共有」ディレクトリをインストールした場所)。

利用可能なテーマのリストを表示するには、`ghostty +list-themes` を実行します。

テーマ ファイルは単純に別の Ghostty 設定ファイルです。同じ構文と同じ設定オプションを共有しています。テーマは有効な設定オプションを設定できるため、信頼できないソースのテーマ ファイルは使用しないでください。組み込みテーマは、安全な設定オプションのみを設定するように監査されています。

テーマ ファイル内で設定できないいくつかのオプションがあります。これらがサポートされていない理由は自明です。テーマ ファイルは `theme` または `config-file` を設定できません。これを書いている時点では、Ghostty はテーマ ファイルでこれらのオプションを設定した場合、警告やエラーを表示しませんが、サイレントに無視します。

背景、前景、パレットなどで指定された追加の色は、テーマで指定された色をオーバーライドします。

ライトモードとダーク モードで異なるテーマを指定するには、次の構文を使用します: `light:theme-name,dark:theme-name`。たとえば: `light:Rose Pine Dawn,dark:Rose Pine`。すべての値の周りの空白はトリミングされ、ライトとダークの順序は重要ではありません。この形式ではライトとダークの両方を指定する必要があります。この形式では、使用されるテーマは現在のデスクトップ環境のテーマに基づきます。

ライト/ダーク モード テーマ設定にはいくつかの既知のバグがあります。これらは将来の更新で修正されます:

- macOS: テーマを切り替えるときにタイトルバー タブ スタイルが更新されません。
