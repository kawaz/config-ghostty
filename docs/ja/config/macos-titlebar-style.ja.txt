# macOS のタイトルバーのスタイル。利用可能な値は "native"、
# "transparent"、"tabs"、"hidden" です。
#
# "native" スタイルはネイティブな macOS タイトルバーをカスタマイズなしで使用します。
# タイトルバーはウィンドウテーマと一致します (`window-theme` を参照)。
#
# "transparent" スタイルは "native" と同じですが、タイトルバーが
# 透明になり、ウィンドウの背景色が透けて見えます。
# これによりウィンドウがより統一された外観になりますが、
# 標準的な macOS アプリケーションより少し異なる見た目になり、
# すべてのテーマで正常に機能しない可能性があります。
#
# "transparent" スタイルはウィンドウ背景色への動的変更に対して
# リアルタイムで更新されます (OSC 11 経由など)。
# より美しい外観にするため、これはターミナルが
# ウィンドウの上部に接するウィンドウ、タブ、またはスプリットの場合にのみ発生します。
# これにより、タイトルバーの色は変わるが最上部のターミナルが
# 一致しないという不連続な外観を避けられます。
#
# "tabs" スタイルはタブバーをタイトルバーに統合した
# 完全にカスタムなタイトルバーです。このタイトルバーは常に
# ターミナルの背景色と一致します。このスタイルにはいくつかの制限があります:
# macOS 13 以下では、保存されたウィンドウの状態がタブを正しく復元しません。
# macOS 14 ではこの問題はなく、他の macOS バージョンはテストされていません。
#
# "hidden" スタイルはタイトルバーを非表示にします。ただし `window-decoration = false` とは異なり、
# ウィンドウのフレームを削除したり、角が丸くなるようにはしません。実行時にこのオプションへの変更や
# このオプションからの変更は、既存のウィンドウに バグのある方法で影響する可能性があります。
#
# "hidden" の場合、最上部のタイトルバー領域はウィンドウのドラッグに
# 使用できなくなります。ウィンドウをドラッグするには、フレームのサイズ変更可能な
# 領域でオプション+クリックを使用してウィンドウをドラッグできます。これは標準的な macOS の動作であり、
# Ghostty が有効にするものではありません。
#
# デフォルト値は "transparent" です。これは見解に基づいた選択ですが、
# 美的に最も優れており、ほとんどの場合で機能すると考えています。
#
# この オプションを実行時に変更すると、新しいウィンドウにのみ適用されます。
macos-titlebar-style = transparent
