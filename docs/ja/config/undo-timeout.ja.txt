# 元に戻すの操作が利用可能なままである期間。この時間の後、
# 操作は元に戻すスタックから削除され、元に戻すことができなくなる。
#
# デフォルト値は 5 秒である。
#
# このタイムアウトは操作ごとに適用される。つまり、複数の操作を実行する場合、
# 各操作は独自のタイムアウトを持つ。新しい操作は前の操作のタイムアウトをリセットしない。
#
# ゼロのタイムアウトは事実上元に戻す操作を無効にする。
# 無限タイムアウトを設定することはできないが、非常に大きなタイムアウトを設定して
# タイムアウトを事実上無効にすることができる（年の単位で）。
# これは、元に戻すスタックが無限に成長し、メモリ使用量が無限に成長し、
# ターミナル セッションが実際には終了しないため、強く推奨されていない。
#
# 期間は、数字の後に時間単位が続く一連の数字として指定される。
# 数字と単位の間にはホワイトスペースが許可される。
# 各数字と単位は合計で総期間を形成するために追加される。
#
# 許可された時間単位は次のとおりである：
#
#   * `y` - 365 SI 日、または 8760 時間、または 31536000 秒。
#     うるう年またはうるう秒の調整は行われない。
#   * `d` - 1 つの SI 日、または 86400 秒。
#   * `h` - 1 時間、または 3600 秒。
#   * `m` - 1 分、または 60 秒。
#   * `s` - 1 秒。
#   * `ms` - 1 ミリ秒、または 0.001 秒。
#   * `us` または `µs` - 1 マイクロ秒、または 0.000001 秒。
#   * `ns` - 1 ナノ秒、または 0.000000001 秒。
#
# 例：
#   * `1h30m`
#   * `45s`
#
# ユニットを繰り返すことができ、加算される。これは
# `1h1h` が `2h` と同等であることを意味する。これは混乱しており、
# 避けるべきである。将来の更新でこれが禁止される可能性がある。
#
# この設定は macOS でのみサポートされている。Linux はまったく元に戻す操作を
# サポートしていないため、この設定は効果がない。
#
# 利用可能：1.2.0 以降
undo-timeout = 5s
