# Ghostty の自動更新機能を制御します。現在、これは macOS でのみサポートされています。
# Linux ビルドは Ghostty によって一元的に制御されていないパッケージマネージャを通じて
# 配布されているためです。
#
# 更新の確認またはダウンロードは、基礎となるプロトコルで義務付けられている標準的な
# ネットワーク情報以外のプロジェクトに情報を送信しません。別の言い方をすれば、
# Ghostty は更新プロセスに明確なトラッキングを追加しません。更新プロセスは最新版に
# 関する情報をダウンロードし、クライアント側で現在のバージョンと比較することで機能します。
#
# 有効な値は:
#
#  * `off` - 自動更新を無効にします。
#  * `check` - 更新を確認し、更新が利用可能な場合はユーザーに通知しますが、自動的に
#    ダウンロードまたはインストールはしません。
#  * `download` - 更新を確認し、自動的に更新をダウンロードし、ユーザーに通知しますが、
#    自動的にインストールはしません。
#
# 設定されていない場合、Sparkle のデフォルト動作に従います。これは標準ユーザーデフォルト
# （`defaults(1)`）に保存されているプリファレンスを尊重します。
#
# 実行時にこの値を変更すると、短い遅延の後に機能します。
