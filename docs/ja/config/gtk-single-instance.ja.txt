# `true` の場合、Ghostty GTK アプリケーションはシングルインスタンスモードで実行されます：
# すでに実行中のプロセスがある場合、起動された新しい `ghostty` プロセスは、
# 新しいウィンドウをもたらします。
#
# `false` の場合、新しい ghostty プロセスごとに別のアプリケーションが起動されます。
#
# `detect` の場合、Ghostty は以下のシナリオのいずれかが見つからない限り、true
# （シングルインスタンス）を想定します：
#
# 1. TERM_PROGRAM 環境変数が空でない値である。この場合、Ghostty がグラフィカル
# ターミナルセッションから起動されていると想定し、専用インスタンスが必要です。
#
# 2. CLI 引数が存在する。この場合、カスタム Ghostty 設定を渡していると想定します。
# シングルインスタンスモードは起動時の設定を継承するため、新しい設定を読み込む
# にはシングルインスタンスを無効にする必要があります。
#
# これらのシナリオのいずれかがいずれかが偽陽性を引き起こしている場合、
# この設定を明示的に設定して、必要な動作を指定できます。
#
# 1.2 より前のオプション `desktop` は非推奨になりました。これを `detect` に置き換えてください。
#
# デフォルト値は `detect` です。
#
# Ghostty のデバッグビルドには別個のシングルインスタンス ID があるため、
# リリースビルドと競合することなくシングルインスタンスをテストできます。
gtk-single-instance = detect
