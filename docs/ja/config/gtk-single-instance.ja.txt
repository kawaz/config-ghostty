# `true` の場合、Ghostty GTK アプリケーションはシングルインスタンスモードで実行されます。
# すでに実行中のプロセスがある場合、起動された新しい `ghostty` プロセスはすべて
# 新しいウィンドウを作成します。
#
# `false` の場合、新しい ghostty プロセスはそれぞれ別々のアプリケーションとして
# 起動されます。
#
# `detect` の場合、以下のシナリオのいずれかが見つからない限り、Ghostty は
# true（シングルインスタンス）であると仮定します。
#
# 1. TERM_PROGRAM 環境変数が空でない値を持つ場合。この場合、
# Ghostty がグラフィカルターミナルセッションから起動されていると
# 判断し、専用のインスタンスが必要だと仮定します。
#
# 2. CLI 引数が存在する場合。この場合、カスタム Ghostty 設定を
# 渡していると仮定します。シングルインスタンスモードは起動時の
# 設定を継承するため、新しい設定を読み込むにはシングルインスタンス
# を無効にする必要があります。
#
# これらのシナリオのいずれかが誤検出を起こしている場合は、
# この設定を明示的に希望する動作に設定することができます。
#
# 1.2 より前のオプション `desktop` は廃止されました。これを `detect` に
# 置き換えてください。
#
# デフォルト値は `detect` です。
#
# Ghostty のデバッグビルドは別のシングルインスタンス ID を持つため、
# リリースビルドと競合せずにシングルインスタンスをテストできます。
gtk-single-instance = detect
