# ランタイムでベルサポートが利用可能な場合に有効にするベル機能。すべての機能
# がすべてのランタイムで利用可能とは限りません。形式はカンマで区切られたベル機能
# のリストです。機能の前に `no-` を付けるとその機能は無効になります。機能を省略
# すると、デフォルト値が使用されます。
#
# 有効な値：
#
#  * `system`
#
#    システムに組み込み関数を使用してユーザーに通知するよう指示します。
#    これにより音声視覚効果、通知、またはその他の動作が実行される可能性があります。
#    これらの効果を変更するには、システム設定を変更する必要があります。
#    たとえば GNOME の「サウンド > アラート音」設定、
#    または KDE Plasma の「アクセシビリティ > システムベル」設定。(GTK のみ)
#
#  * `audio`
#
#    カスタム音声を再生します。(GTK のみ)
#
#  * `attention` *(デフォルトで有効)*
#
#    Ghostty がフォーカスされていないときにユーザーの注意を促し、再度フォーカス
#    を受けるまで継続します。macOS では、アプリアイコンがドック内で 1 回バウンス
#    します。Linux では、動作はデスクトップ環境やウィンドウマネージャー/コンポーザ
#    に依存します：
#
#    - KDE では、タスクバー内のデスクトップアイコンの背景がハイライトされます；
#
#    - GNOME では、クリックすると Ghostty ウィンドウをフォーカスに戻す通知を受け
#      取る可能性があります；
#
#    - Sway では、ウィンドウが明確に色分けされた境界線で装飾される場合があります；
#
#    - その他のシステムではこれは何の効果もない場合があります。
#
#  * `title` *(デフォルトで有効)*
#
#    アラート表面のタイトルの前にベル絵文字 (🔔) を付加します。ターミナルが再
#    フォーカスされるか、(キーボード入力などで) 操作されるまで継続します。
#
#  * `border`
#
#    アラート表面の周囲に枠線を表示します。ターミナルが再フォーカスされるか、
#    (キーボード入力などで) 操作されるまで継続します。
#
#    GTK のみ。
#
# 例：`audio`、`no-audio`、`system`、`no-system`
#
# 利用開始時期：1.2.0
bell-features = no-system,no-audio,attention,title,no-border
