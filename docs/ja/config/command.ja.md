# command

**Language:** [English](../../en/config/command.en.md) | 日本語

## Description

実行するコマンド。通常はシェルです。これが絶対パスでない場合は、`PATH` で検索されます。これが設定されていない場合、デフォルトはシステムから検索されます。デフォルト検索のルールは:

- `SHELL` 環境変数
- `passwd` エントリ (ユーザー情報)

これには、コマンドを実行するための追加の引数を含めることができます。追加の引数が指定されている場合、コマンドはシェル引数の展開をオフロードするために `/bin/sh -c` を使用して実行されます。

シェル展開を完全に回避するには、コマンドに `direct:` というプレフィックスを付けます。たとえば、`direct:nvim foo`。これにより `/bin/sh` への往復が回避されますが、スペース付きの引数、`~` 付きのファイルパス、グローブなどのシェル解析もサポートされません。(利用可能: 1.2.0 以降)

コマンドに明示的に `shell:` というプレフィックスを付けて、コマンドを常にシェルでラップすることもできます。これは、正しいモードを選択するためのヒューリスティックが使用されていない場合に、ヒューリスティックが間違っていないことを確認するために使用できます。(利用可能: 1.2.0 以降)

このコマンドはすべての新しいターミナル サーフェス、つまり新しいウィンドウ、タブなどに使用されます。Ghostty の起動時に作成された最初のターミナル サーフェスに対してのみコマンドを実行したい場合は、`initial-command` 設定を使用します。

Ghostty は、引数を使用してコマンドを実行するための一般的な `-e` フラグをサポートしています。たとえば、`ghostty -e fish --with --custom --args`。このフラグは `initial-command` 設定を設定します。詳細については、それを参照してください。
