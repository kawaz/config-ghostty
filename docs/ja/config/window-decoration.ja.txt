# ウィンドウ装飾(window decoration)の設定を行います。この設定は
# _preference_ (優先設定)を指定するもので、実際の OS、デスクトップ環境、
# ウィンドウマネージャなどがこの優先設定をオーバーライドする可能性があります。
# Ghostty はこの優先設定を尊重するために最善の努力をしていますが、
# 常に可能とは限りません。
#
# 有効な値:
#
#  * `none`
#
#    すべてのウィンドウ装飾が無効になります。タイトルバー、ボーダーなど
#    は表示されません。macOS では、システムにより強制的にタブも無効に
#    なります。
#
#  * `auto`
#
#    現在の OS およびデスクトップ環境の検出された優先設定に基づいて、
#    クライアント側またはサーバー側の装飾のいずれかを使用するかを自動的に
#    決定します。このオプションは通常、Ghostty をデスクトップに最も
#    「ネイティブ」な外観にします。
#
#  * `client`
#
#    クライアント側の装飾を優先します。
#
#    利用可能: 1.1.0 以降
#
#  * `server`
#
#    サーバー側の装飾を優先します。これは Linux 上の GTK でのみ関連しており、
#    X11 上、または `org_kde_kwin_server_decoration` プロトコルをサポートする
#    コンポジタ上の Wayland (例: KDE Plasma、ほぼすべての GNOME 以外の
#    デスクトップがこのプロトコルをサポート)のいずれかです。
#
#    `server` が設定されていますが、環境がサーバー側の装飾をサポートしていない
#    場合は、クライアント側の装飾が代わりに使用されます。
#
#    利用可能: 1.1.0 以降
#
# デフォルト値は `auto` です。
#
# 後方互換性と利便性のために、この設定は boolean 値 (true と false)
# も受け入れます。`true` に設定された場合、これは `auto` と同等です。
# `false` に設定された場合、これは `none` と同等です。
# これは、クライアント側とサーバー側の装飾を区別しないシステム上で
# 主に生活するユーザーにとって便利です(例: macOS および Windows)。
#
# "toggle_window_decorations" キーバインドアクション(keybind action)は、
# この設定を実行時に切り替えるためのキーバインディングを作成するために
# 使用できます。
#
# macOS: ネイティブウィンドウボーダーまたは角の丸みを削除せずにタイトルバーを
#        非表示にするには、代わりに `macos-titlebar-style = hidden` を
#        使用してください。
window-decoration = auto
