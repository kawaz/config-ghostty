# `PATH` で検索されます。これが設定されていない場合、デフォルトはシステムから
# 検索されます。デフォルト検索のルールは:
#
#   * `SHELL` 環境変数
#
#   * `passwd` エントリ (ユーザー情報)
#
# これには、コマンドを実行するための追加の引数を含めることができます。
# 追加の引数が指定されている場合、コマンドはシェル引数の展開を
# オフロードするために `/bin/sh -c` を使用して実行されます。
#
# シェル展開を完全に回避するには、コマンドに `direct:` というプレフィックスを付けます。
# たとえば、`direct:nvim foo`。これにより `/bin/sh` への往復が回避されますが、
# スペース付きの引数、`~` 付きのファイルパス、グローブなどのシェル解析も
# サポートされません。(利用可能: 1.2.0 以降)
#
# コマンドに明示的に `shell:` というプレフィックスを付けて、コマンドを常にシェルで
# ラップすることもできます。これは、正しいモードを選択するためのヒューリスティックが
# 使用されていない場合に、ヒューリスティックが間違っていないことを確認するために使用できます。
# (利用可能: 1.2.0 以降)
