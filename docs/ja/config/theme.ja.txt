# 使用するテーマ。これは組み込みのテーマ名、カスタムテーマ名、または
# カスタムテーマファイルの絶対パスを指定できます。Ghostty はライトモード
# とダークモードで異なるテーマを指定することもサポートしています。各オプション
# については以下に記載されています。
#
# テーマが絶対パス名の場合、Ghostty はそのファイルをテーマとして
# ロードしようとします。そのファイルが存在しない場合または
# アクセスできない場合、エラーが記録され、他のディレクトリは
# 検索されません。
#
# テーマが絶対パス名ではない場合、テーマ名と一致するファイル名を探すために
# 2つの異なるディレクトリが検索されます。これはケースセンシティブな
# ファイルシステムを持つシステムではケース区別があります。テーマ名に
# パスセパレータが含まれている場合はエラーになります。ただし、絶対パス名
# である場合は除きます。
#
# 最初のディレクトリは、Ghostty 設定ディレクトリの `themes` サブディレクトリです。
# これは `$XDG_CONFIG_HOME/ghostty/themes` または `~/.config/ghostty/themes` です。
#
# 2番目のディレクトリは、Ghostty リソースディレクトリの `themes` サブディレクトリです。
# Ghostty は多数のテーマを含めてシップされており、このディレクトリに
# インストールされます。macOS では、これは
# `Ghostty.app/Contents/Resources/ghostty/themes` ディレクトリにあります。
# Linux では、`share/ghostty/themes` ディレクトリにあります（Ghostty の
# "share" ディレクトリをインストールした場所に応じます）。
#
# 利用可能なテーマのリストを表示するには、`ghostty +list-themes` を実行してください。
#
# テーマファイルは単なる別の Ghostty 設定ファイルです。それらは同じ構文と
# 同じ設定オプションを共有しています。テーマは有効な設定オプションを
# 設定できるため、信頼できないソースのテーマファイルは使用しないでください。
# 組み込みテーマは安全な設定オプションのみを設定するよう監査されています。
#
# テーマファイル内で設定できないオプションもあります。これらが
# サポートされていない理由は明白なはずです。テーマファイルは `theme` または
# `config-file` を設定することはできません。この執筆時点では、Ghostty は
# テーマファイルでこれらのオプションを設定しても警告またはエラーを表示しませんが、
# 静かに無視されます。
#
# background、foreground、palette などで指定された追加の色は、
# テーマで指定された色をオーバーライドします。
#
# ライトモードとダークモードで異なるテーマを指定するには、次の構文を使用します：
# `light:theme-name,dark:theme-name`。例：
# `light:Rose Pine Dawn,dark:Rose Pine`。すべての値の前後の空白文字は
# トリミングされ、light と dark の順序は関係ありません。light と dark の両方が
# この形式で指定される必要があります。この形式では、使用されるテーマは
# 現在のデスクトップ環境のテーマに基づいています。
#
# ライト/ダークモードのテーマ設定にはいくつかの既知の問題があります。
# これらは将来のアップデートで修正される予定です：
#
