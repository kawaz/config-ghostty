# 読み込む追加設定ファイル。この設定は繰り返し指定して
# 複数の設定ファイルを読み込むことができます。設定ファイル自体が
# さらに別の設定ファイルを読み込むこともできます。パスは
# `config-file` ディレクティブを含むファイルからの相対パスです。
# コマンドライン引数の場合、パスはカレントワーキングディレクトリ
# からの相対パスになります。
#
# ファイルパスの先頭に ? 文字を付けると、ファイルが存在しない
# 場合のエラーを抑制します。リテラルな ? 文字で始まるファイルを
# インクルードしたい場合は、ファイルパスを二重引用符（"）で
# 囲んでください。
#
# サイクル（循環参照）は許可されていません。サイクルが検出された
# 場合、エラーがログに出力され、設定ファイルは無視されます。
#
# 設定ファイルは、それらが定義されている設定の後に、定義されている
# 順序で読み込まれます。**これは非常に微妙ですが重要な点です。**
# 別の言い方をすれば：設定ファイルは設定全体が読み込まれた後まで
# 有効にはなりません。たとえば、以下の設定では：
#
# ```
# config-file = "foo"
# a = 1
# ```
#
# 「foo」が `a = 2` を含む場合、`a` の最終値は 2 になります。
# なぜなら、`foo` は `config-file` 値を設定する設定ファイルの
# 後に読み込まれるためです。
