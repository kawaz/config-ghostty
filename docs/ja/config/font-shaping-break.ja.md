# font-shaping-break

**言語:** [English](../../en/config/font-shaping-break.en.md) | 日本語

## 説明

フォント シェーピングを複数の実行に分割する場所を制御します。

「実行」は、一緒に形成されるテキストの隣接するセグメントです。「シェーピング」は、テキスト（コードポイント）をグリフ（レンダリング可能な文字）に変換するプロセスです。これはリガチャがどのように形成されるかです。たとえば、コーディング フォントが「!=」を単一のグリフに変えた場合、「!」と「=」が単一の実行で隣同士にいることを確認する必要があります。実行が破損すると、テキストは個別に形成されます。例を続けると、「!」が 1 つの実行の最後にあり、「=」が次の実行の開始にある場合、リガチャは形成されません。

Ghostty は可読性またはユーザビリティを改善するために、特定の点で実行を破損します。たとえば、Ghostty はデフォルトでカーソルの下で実行を破損して、テキスト編集がリガチャではなく個々の文字を確認できるようにします。この設定により、この動作を構成することができます。

複数のオプションを設定するには、コンマで値を結合します。オプションを無効にするには、「no-」をプレフィックスに付けます。オプションの有効化と無効化は同時に行うことができます。

利用可能なオプション：

- `cursor` - カーソルの下で実行を破損します。

利用可能：1.2.0 以降

## デフォルト値

```conf
# https://ghostty.org/docs/config/reference#font-shaping-break
# テキスト レンダリング用のフォント シェーピング分割ポイントを制御 (default: cursor)
font-shaping-break = cursor
```
